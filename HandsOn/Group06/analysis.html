Data Analysis

We have decided which of the resource's attributes we will keep for our project.

Our decision was to remove the dea code of the defibrilators as it is not relevant to our application, as well as the district code

We will also filter if the defibrilator's location is a public one by filtering the ownership column, which has a range of "Public" and "Private"

Licensing

The licensing of the data source is (CC BY 4.0), which means it is okay to use it commercialy and change the licensing as long as we give credit to the author. We do not plan to change the licensing as of yet.

Naming strategy

The naming strategy for the project will include an URI for each defibrilator

http://desfibriladores.com/resource/defibrilador/112

The attributes location_direction and the direction of the defibrilator will redirect the user to the web page of the location and the Google Maps URI of the coordinates respectively

Structure of the ontology

We have decided to create the following classes and properties to organize our ontology:

  • Classes: Desfibrilador, Establecimiento, Titularidad, Municipio, Direccion, HorarioAcceso
  • Properties: hasEstablecimiento, hasTitularidad, hasCodigoMunicipio, hasNombreMunicipio, hasCodigoViaDireccion, hasNombreViaDireccion, hasNumeroPortalDireccion, hasPisoDireccion, hasPuertaDireccion, hasCodigoPostalDireccion, hasUbicacionDireccion, hasCoordXDireccion, hasCoordYDireccion, hasHorarioAcceso, hasMunicipio, hasDireccion.
